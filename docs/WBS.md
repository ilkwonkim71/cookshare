# CookShare MVP 개발 WBS (Work Breakdown Structure)

레시피 공유 서비스 MVP 개발의 작업 분해 구조입니다.

- **프로젝트 기간**: 3개월 (12주) — 2026-06-22 ~ 2026-09-11
- **팀 구성**: 풀스택 개발자 2명 (Dev A, Dev B)
- **스프린트**: 2주 단위 × 6 스프린트
- **기준 명세**: `docs/MVP-SPEC.md`

## 가정 및 산정 기준

- 1인 1주 가용 공수 = **30h** (회의/리뷰/컨텍스트 스위칭 제외한 순수 개발 시간)
- 2인 1주 팀 공수 = **60h**, 12주 총 팀 공수 ≈ **720h**
- 스캐폴딩(인증·CRUD·업로드 기반 코드)은 완료된 상태 → WBS는 **완성/보강/신규** 기준으로 산정
- 시간 단위: h(시간), d(일=6h)

## 역할 분담 원칙

- **Dev A**: 백엔드 중심 (API, DB, 인증, 보안, 인프라) + FE 일부
- **Dev B**: 프론트엔드 중심 (UI, 화면, 상태관리, 디자인 시스템) + BE 일부
- 두 명 모두 풀스택이므로 스프린트별로 부하에 따라 교차 지원
